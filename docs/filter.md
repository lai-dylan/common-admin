# Element Plus通用表格过滤器支持功能清单

基于Element Plus封装通用表格时，表格上方过滤器需兼顾「易用性、灵活性、复用性」，贴合Element Plus组件生态，同时覆盖绝大多数业务场景。以下按「核心必备功能」「体验优化功能」「高级拓展功能」三个层级梳理，每个功能均对应Element Plus相关组件，便于直接落地封装。

# 一、 核心必备功能（满足80%的常规业务场景）

这是通用表格过滤器的基础，必须支持，且要做到配置化、低耦合。

## 1.  文本类筛选（模糊/精确）

针对表格中的「字符串类型字段」（如姓名、编号、备注），是最常用的筛选场景。

- 支持 **模糊匹配**（默认，关键词包含匹配，如输入“张”匹配“张三”“张伟”）

- 支持 **精确匹配**（可选配置，如输入“001”仅匹配编号“001”，不匹配“0011”）

- 对应Element Plus组件：`ElInput`（单行输入），可选支持`ElInput(type="textarea")`（长文本备注筛选）

- 附加细节：输入完成后「回车触发」「失焦触发」，或配合“查询”按钮触发，避免实时输入频繁刷新表格（可配置防抖，如300ms）。

## 2.  枚举类筛选（单选/多选）

针对表格中的「固定枚举值字段」（如状态、性别、角色、类型），避免用户手动输入出错，提升筛选精准度。

- 下拉单选：适用于“互斥性”枚举（如性别：男/女；状态：启用/禁用），对应`ElSelect + ElOption`

- 下拉多选：适用于“并列性”枚举（如角色：管理员/普通用户/访客；标签：热门/新品/折扣），对应`ElSelect(multiple) + ElOption`

- 附加细节：


    - 多选支持「全选/取消全选」（可选，提升批量筛选效率）

    - 支持「默认选中项」「清空选项」功能

    - 枚举选项支持配置化传入（如`options: [{ label: '启用', value: 1 }, { label: '禁用', value: 0 }]`）

## 3.  数值类筛选（单个/区间）

针对表格中的「数值类型字段」（如金额、年龄、数量、ID）。

- 单个数值筛选：支持「大于」「小于」「等于」「不等于」（可选配置匹配规则），对应`ElInputNumber`或`ElInput(type="number")`

- 数值区间筛选：最常用（如“金额100-1000”“年龄20-30”），对应两个`ElInputNumber`或`ElInput(type="number")`（左侧“最小值”，右侧“最大值”），逻辑为「大于等于最小值 且 小于等于最大值」

- 附加细节：支持空值处理（如仅填最小值，即为“大于等于该值”；仅填最大值，即为“小于等于该值”）

## 4.  日期/时间类筛选（单个/区间）

针对表格中的「时间类型字段」（如创建时间、更新时间、下单日期），是业务系统的高频筛选场景。

- 单个日期：筛选某一天的数据（如“2024-02-04”），对应`ElDatePicker(type="date")`

- 日期区间：最常用（如“2024-01-01 至 2024-01-31”），对应`ElDatePicker(type="daterange")`

- 时间区间：精准到时分秒（如“2024-02-04 09:00:00 至 2024-02-04 18:00:00”），对应`ElDatePicker(type="datetimerange")`

- 附加细节：


    - 内置快捷选项（今日、昨日、近7天、近30天、本月、上月、本年），Element Plus的`ElDatePicker`原生支持，封装时直接配置即可，极大提升用户体验

    - 支持日期格式自定义（如`YYYY-MM-DD` `YYYY-MM-DD HH:mm:ss`）

## 5.  基础交互按钮（查询/重置）

过滤器的必备操作入口，保证用户能主动触发筛选和清空条件。

- 「查询」按钮：点击后收集所有筛选条件，触发表格数据刷新（本地筛选/远程接口筛选），对应`ElButton`

- 「重置」按钮：一键清空所有筛选项的输入/选中状态，恢复表格初始数据，对应`ElButton`（建议设置为文字按钮，样式更简洁）

## 6.  筛选项配置化

作为「通用表格封装」的核心，过滤器的所有筛选项必须支持**配置化定义**，而非硬编码，这样才能实现复用。

- 每个筛选项通过一个配置对象描述，核心属性至少包含：

```javascript

{
  label: '姓名', // 筛选项标签文字
  prop: 'name', // 对应表格数据的字段名（用于收集筛选值）
  type: 'input', // 筛选组件类型（input/select/date/number等）
  placeholder: '请输入姓名', // 占位符
  options: [], // 仅select类型需要，枚举选项数组
  isRange: false, // 仅number/date类型需要，是否为区间筛选
  disabled: false, // 是否禁用该筛选项
  // 其他扩展属性（如模糊匹配、日期格式等）
}
    
```

封装时通过遍历筛选项配置数组，自动渲染对应的筛选组件，无需修改模板。

# 二、 体验优化功能（提升用户使用感，适配复杂场景）

这类功能不是必须，但能显著提升过滤器的易用性，让通用表格更贴合实际业务。

## 1.  筛选项折叠/展开

当筛选项较多（如超过6个）时，默认折叠部分不常用的筛选项，仅展示核心筛选项，避免页面拥挤。

- 提供「展开更多」/「收起」按钮，点击切换筛选项的显示/隐藏状态

- 建议记录用户的折叠状态（临时缓存），刷新后保持一致

## 2.  筛选条件回显与记忆

- 筛选后，保留用户的所有筛选条件（输入框内容、下拉选中项、日期区间等），不会因为表格分页切换、刷新页面（可选）而丢失

- 进阶：支持基于`localStorage/sessionStorage`记忆筛选条件，用户再次进入该页面时，自动恢复上次的筛选状态（需配置是否开启）

## 3.  筛选项联动

支持筛选项之间的「数据联动」，适用于有层级关系的字段（如省-市-区、部门-小组、商品分类-商品列表）。

- 示例：选择“省份”后，“城市”筛选项的下拉选项自动加载该省份对应的城市数据（可本地联动，也可远程接口获取）

- 对应Element Plus组件：`ElSelect` + 联动接口请求，或`ElCascader`（级联选择器，适用于层级明确且无需单独展示每个层级的场景）

## 4.  防抖筛选（针对即时触发场景）

如果配置了「输入后自动触发筛选」（如文本输入、下拉选择后立即刷新表格），必须添加**防抖处理**（如300-500ms），避免用户快速输入或频繁切换时，触发大量无效的表格刷新（尤其是远程接口筛选场景）。

## 5.  与表格分页联动

筛选条件变更并触发查询后，表格自动重置到「第1页」，避免出现“筛选后只有5条数据，却停留在第10页”的尴尬场景。

## 6.  筛选项布局适配

支持「行内紧凑布局」（默认，筛选项横向排列，自动换行）和「独占行布局」（每个筛选项独占一行，适用于复杂筛选项），可通过配置切换，适配不同屏幕尺寸和业务需求。

# 三、 高级拓展功能（覆盖小众复杂场景，提升封装的通用性上限）

这类功能针对特殊业务场景，通用表格封装可提供扩展入口，无需强制实现。

## 1.  自定义筛选组件

通用筛选组件无法覆盖所有场景（如树形选择、图片上传筛选、多级联动筛选），因此需要支持「传入自定义组件」的能力。

- 预留插槽（Slot）或自定义渲染函数，允许外部传入自定义筛选组件，并能将组件的筛选值同步到通用表格的筛选条件中

- 示例：支持传入`ElTreeSelect`（树形选择器），用于筛选“部门树形结构”数据

## 2.  模糊匹配模式可配置

文本筛选默认是「包含匹配」，进阶可支持配置匹配模式：

- 开头匹配（如输入“张”，仅匹配“张三”，不匹配“张三丰”）

- 结尾匹配（如输入“三”，仅匹配“张三”，不匹配“张三丰”）

- 全匹配（即精确匹配，与数值筛选的“等于”一致）

## 3.  批量筛选模板保存

针对高频重复筛选场景（如“每月筛选上月已付款且金额大于1万的订单”），支持用户将当前筛选条件保存为「筛选模板」，下次可直接加载模板，无需重新设置。

## 4.  本地筛选/远程筛选切换

- 本地筛选：表格数据已全部加载到前端，筛选时直接在前端处理数据（无需请求接口），适用于数据量较小（如小于1000条）的场景，对应Element Plus表格的原生过滤能力

- 远程筛选：筛选条件传递给后端，由后端返回筛选后的结果，适用于大数据量场景（如大于1000条）

- 封装时支持通过配置项切换筛选模式，且提供统一的筛选结果回调，保证上层使用无感知

# 总结

1. 通用表格过滤器的核心是「**配置化**」和「**覆盖高频场景**」，优先实现文本、枚举、数值、日期筛选及查询/重置按钮。

2. 体验优化功能（折叠、回显、联动、防抖）是提升用户使用感的关键，也是通用封装区别于硬编码表格的核心优势。

3. 高级拓展功能（自定义组件、本地/远程切换）可根据业务需求选择性实现，提升封装的通用性和上限。

按照这个功能清单封装，就能得到一个适配大多数业务场景、易用且灵活的Element Plus通用表格过滤器。
