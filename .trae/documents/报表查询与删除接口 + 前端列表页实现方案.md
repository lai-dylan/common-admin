**后端实现**
- 新增路由文件 server/routes/reports.js 并在 app.js 挂载：
  - GET /api/reports
    - 接收 query：page, pageSize, sortProp, sortOrder
    - 接收筛选：
      - keyword（文本包含匹配 reportName）
      - minVisits（数字输入，按 >= 过滤 visits）
      - reportType（下拉单选）
      - department（下拉单选）
      - relatedSystems（下拉多选，数组，包含任意匹配）
      - enabled（switch 布尔）
    - 在内存 reports 上按筛选过滤 → 排序 → 分页，返回 { rows, total }
  - POST /api/reports/delete
    - 请求体 { ids: number[] }
    - 在内存 reports 数组中删除匹配 id，返回删除数量
  - 统一响应使用 res(ctx, data, message)

**前端实现**
- 新增 API 文件 client/src/api/reports.ts：
  - getReports(params)：将 CommonTable 的 QueryPayload 映射为服务端 query 参数
  - deleteReports(ids)：POST 删除
- 新增页面 client/src/views/reports/index.vue：
  - 使用 CommonTable：
    - columnConfigs：展示 reportId, reportName, reportType, department, owner, dataPeriod, enabled, published, visits, qualityScore, relatedSystems
    - filterConfigs：满足“文字输入、数字输入、下拉单选、下拉多选、switch”五类：
      - keyword: kind='input', placeholder='报表名称关键字'
      - minVisits: kind='component' + el-input-number，normalize 转为数字
      - reportType: kind='select'（静态 options 或加载）
      - relatedSystems: kind='multi-select'（静态 options 或加载）
      - enabled: kind='checkbox' 作为开关（或自定义 component switch）
    - fetcher：调用 getReports，将前端 Filters 映射为服务端参数；处理分页与服务端排序
    - tableActions：批量删除（dangerButton），对 selectedKeys 执行 deleteReports → refresh
    - rowActions：单行删除，执行 deleteReports([row.reportId]) → refresh
  - autoQuery: true，支持 URL q 同步
- 路由与菜单：在 client/src/router/index.ts 的主布局 children 中新增：
  - path: '/reports', name: 'Reports', meta: { title: '报表管理', icon: 'Document' }

**参数映射约定**
- CommonTable QueryPayload → 服务端：
  - pagination.page → page, pagination.pageSize → pageSize
  - sort.prop → sortProp, sort.order → sortOrder（'asc'|'desc'）
  - filters：
    - keyword → keyword
    - minVisits → minVisits
    - reportType → reportType
    - relatedSystems → relatedSystems[]（brackets 序列化）
    - enabled → enabled（true/false）
- 服务器返回：{ code:200, data:{ rows: Report[], total:number }, message:'success' }
